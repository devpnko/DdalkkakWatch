C1 — Mac 내장 마이크 + 워치 PTT 검증

가장 빠른 셋업. 추가 장비 0, 3분 안에 받아쓰기 가능. Mac 1m 이내일 때 가장 좋아.

1) 워치 개발자 모드 + 무선 디버깅 켜기

워치 앱은 현재 adb 사이드로드로 설치 (Galaxy Store 미등록). 워치에서:

  1. 설정 → 워치 정보 → 소프트웨어 정보 → "빌드 번호" 7번 연속 탭 → 개발자 모드 ON
  2. 설정 → 개발자 옵션 → 무선 디버깅 ON
  3. 무선 디버깅 → "디버깅 허용 기기에서 페어링" → IP:포트 + 6자리 페어링 코드 메모
  4. 같은 화면 위쪽의 IP 주소 : 포트(연결용, 페어링 포트와 다름)도 메모

2) Mac에서 워치에 앱 설치 (adb)

⚠️ 이건 개발 설치용 adb 페어링 — 아래 4)의 BT 페어링(실사용)과 다릅니다.

# Mac terminal — 1)에서 메모한 값 사용
adb pair 192.168.x.x:페어링포트        # 6자리 코드 입력
adb connect 192.168.x.x:연결포트
adb devices                            # 워치가 device로 보여야 OK

# APK 설치 (릴리스 받았으면)
adb install -r DdalkkakWatch.apk
# 또는 소스 빌드:
./gradlew :app:assembleDebug && \
  adb install -r app/build/outputs/apk/debug/app-debug.apk

워치가 sleep으로 끊기면 화면 깨우고 adb connect 재시도. (자동 재연결 데몬은 README 참고)

3) 받아쓰기 단축키 설정 (Mac)

OpenTypeless / Superwhisper / macOS 받아쓰기 중 택일 → 단축키를 Opt+Cmd+X로.

Spotlight과 충돌 안 나게 Opt+Cmd+X 권장.

4) 워치 ↔ Mac 블루투스 페어링 (실사용 연결)

⚠️ 2)의 adb와 별개 — 이게 실제 키 입력이 흐르는 BT HID 연결입니다.

  1. 워치에서 딸깍 앱 실행 → 화면 한 번 탭 → "2분간 Discoverable".
  2. Mac → 시스템 설정 → Bluetooth → DdalkkakWatch 찾기 → 연결.
  3. 워치에 페어링 코드 표시되면 동일 코드 확인 후 수락.
  4. 워치 상태가 "누름=Typeless · 더블탭=Enter"로 바뀌면 OK.

5) Mac 마이크 입력 확인

  1. 시스템 설정 → 사운드 → 입력 → MacBook Pro 마이크 (또는 외장) 선택
  2. "입력 레벨" 막대가 움직이는지 한 번 발화로 확인

6) 검증 발화

  1. 아무 텍스트 필드 (메모/Cursor/Claude Code)에 커서
  2. 워치 화면 꾹 누름 (200ms+) → 받아쓰기 UI 활성화
  3. "안녕하세요" 발화 → 손 떼면 텍스트 입력
  4. 짧게 두 번 탭 = Enter

7) (선택) 화면도 안 켜고 — Knock-Knock

여기까지 됐으면, 손목 두 번 두드림만으로 화면 없이 받아쓰기를 켤 수 있어요.

# Security Policy

## Supported Versions

Security fixes target the default branch until public versioned releases exist.
After the first tagged release, supported release lines should be listed here.

## Reporting a Vulnerability

Use GitHub Security Advisories if they are enabled for this repository.

If private advisories are not available yet, open a minimal public issue that
asks for a private disclosure channel. Do not include exploit steps, Bluetooth
addresses, device identifiers, personal logs, or private screenshots in a public
issue.

Useful private report details:

- Watch model and Wear OS version.
- macOS version and dictation app.
- App version or commit SHA.
- Reproduction steps.
- Relevant `adb logcat` lines after removing device identifiers.

The project should acknowledge valid reports before publishing details or fixes.
